Chapter 7 Terminology

Terms Defintions
Software Engineering Software engineering is the process of designing, creating, testing, and maintaining software. It involves applying principles and techniques from computer science, mathematics, and engineering to develop high-quality software that meets the needs of users.
Integrated Development Environment (IDE) An integrated development environment (IDE) is a software application that provides developers with a comprehensive set of tools for writing, testing, and debugging code. IDEs typically include a code editor, a debugger, and a build automation tool, among other features.
Association for Computing Machinery (ACM) An assembler is a program that converts assembly language code into machine language code.
Assembly Language The Association for Computing Machinery (ACM) is an international professional organization for computer scientists and computing professionals. The purpose of the ACM is to advance computing as a science and a profession, and to promote the development of high-quality computing education.
Benefits of ACM Membership Students can join ACM at a reduced rate and receive the same benefits as regular members. Student members can also participate in programming competitions and other educational events organized by ACM
Institute of Electrical and Electronics Engineers (IEEE) The Institute of Electrical and Electronics Engineers (IEEE) is a professional organization for electrical engineers, computer scientists, and other technology professionals. The purpose of the IEEE is to promote the advancement of technology for the benefit of humanity.
Benefits of IEEE Membership IEEE membership provides access to a range of resources, including technical publications, conferences, and networking opportunities. Members can also participate in standards development and other professional activities.
Student IEEE Membership Students can join IEEE at a reduced rate and receive the same benefits as regular members. Student members can also participate in student competitions, conferences, and other educational events organized by IEEE.
Software Life Cycle: Development Phase The development phase of the software life cycle involves four main stages:
  • Requirements analysis: The process of gathering and analyzing user needs and requirements for the software.
  • Design: The process of creating a blueprint or plan for how the software will be built.
  • Implementation: The process of writing the actual code for the software.
  • Testing: The process of verifying that the software works as intended and meets the user's requirements.
Modular Implementation Modular implementation is a software development approach that involves breaking down a software system into smaller, more manageable modules or components. Each module is designed and developed independently, and can be tested and debugged separately from the rest of the system. This approach can lead to more maintainable, scalable, and reusable code.
Unified Modeling Language (UML) Unified Modeling Language (UML) is a graphical notation used to describe and visualize the design of a software system. UML provides a standardized way to represent different aspects of a software system, including its structure, behavior, and interactions with other systems.
Use Case A use case is a description of a particular interaction between a user (or actor) and a software system. A use case describes what the user wants to accomplish, and how the software system should respond to the user's actions.
Data Relationships:
  • One-to-one: A one-to-one relationship is a data relationship in which each record in one table is associated with at most one record in another table. For example, in a database of employees and their office locations, each employee can have only one office location, and each office location can be associated with only one employee.
  • One-to-many: A one-to-many relationship is a data relationship in which each record in one table can be associated with multiple records in another table. For example, in a database of customers and their orders, each customer can have multiple orders, but each order can be associated with only one customer.
  • Many-to-many: A many-to-many relationship is a data relationship in which each record in one table can be associated with multiple records in another table, and vice versa. For example, in a database of students and their courses, each student can take multiple courses, and each course can be taken by multiple students.
Software Testing:
  • Glass Box Testing: Glass box testing is a testing technique that involves examining the internal structure of a software system. This technique is also known as white box testing, and is typically performed by developers or quality assurance engineers who have access to the source code. Glass box testing can help identify defects and potential performance issues in the software.
  • Black Box Testing: Black box testing is a testing technique that involves testing a software system without examining its internal structure. This technique is also known as functional testing, and is typically performed by quality assurance engineers who do not have access to the source code. Black box testing focuses on the behavior of the software and can help ensure that it meets the user's requirements.
  • Beta Testing: Beta testing is a testing phase in which the software is tested by a group of users outside of the development team. Beta testers typically use the software in a real-world setting and provide feedback on its usability, performance, and functionality. Beta testing can help identify defects and usability issues that may not have been caught during earlier testing phases.
  • Alpha Testing: Alpha testing is a testing phase in which the software is tested by members of the development team before it is released to beta testers or the general public. Alpha testing can help identify defects and usability issues that may not have been caught during earlier development phases.
Documentation:
  • User Documentation: User documentation is a type of documentation that provides information to users about how to use a software product. User documentation can include user guides, tutorials, and other instructional materials.
  • System Documentation: System documentation is a type of documentation that provides information to developers and other technical staff about the design and implementation of a software system. System documentation can include technical specifications, design documents, and other technical reference materials.
  • Technical Documentation: Technical documentation is a type of documentation that provides detailed technical information about a software product, including specifications, code documentation, and other reference materials.

    Intellectual Property Law:
    • Software License: A software license is a legal agreement between the software developer and the user that defines how the software can be used. Software licenses can specify the terms of use, distribution, modification, and other aspects of software usage.
    • Software Disclaimer: A software disclaimer is a statement that disclaims or limits the liability of the software developer for any damages that may arise from the use of the software. A disclaimer can be included in a software license agreement or displayed to users when they first use the software.
    Best Defenses Against Software Liability Claims: The best defenses against software liability claims are to ensure that the software is designed and developed with the highest level of quality and to provide clear and accurate documentation and instructions for use.
